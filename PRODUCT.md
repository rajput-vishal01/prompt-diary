# Product

## Register

product

## Users

Developers and AI power-users who live in ChatGPT/Claude. Context: mid-flow in a chat, they open the extension popup for ~5 seconds to grab or save a prompt, then leave. Dashboard is the slower "organize my library" surface. They already trust tools like Linear, Raycast, Stripe.

## Product Purpose

A password-manager-style vault for prompts. Great prompts get lost in chat history; Prompt Diary captures them (context menu / popup), organizes them (folders, tags), syncs them, and shares them (private / team / public). Success = the popup becomes muscle memory: select → save; open → copy.

## Brand Personality

Precise, archival, calm. A well-kept ledger, not a neon dashboard. The tool disappears into the task.

## Anti-references

- The 2025 AI-gen default: near-black `#0f1117` background + indigo/violet `#6366f1` accent + glassmorphism. This is exactly what we're moving away from.
- Hacker-terminal green-on-black. The second-order reflex for "dev tool that isn't indigo".
- SaaS gradient-hero marketing chrome.

## Design Principles

1. **Ledger, not dashboard** — light paper-neutral surfaces, hairline rules, rows not card grids. Data reads like entries in a book.
2. **Prompts are artifacts** — prompt bodies always render in monospace; titles and UI in the sans. The content is the hero.
3. **One ink** — bottle green carries brand, primary actions, and selection. Nothing else gets saturated color except semantic state (team = amber, danger = oxblood).
4. **Five-second popup** — extension surface optimizes for instant scan + one-click copy. Density over decoration.
5. **Earned familiarity** — standard affordances (buttons, selects, modals) done precisely; no invented controls.

## Accessibility & Inclusion

WCAG AA: body text ≥4.5:1, focus rings visible everywhere, keyboard operable, `prefers-reduced-motion` respected, color never the only signal (visibility badges have labels).
